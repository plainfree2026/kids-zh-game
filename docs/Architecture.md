## 2. 最终选型结论及技术约束

### 2.1 选型结论

| 决策项 | 选型 | 核心理由 |
|-------|------|---------|
| **前端框架** | 微信小程序原生开发 | 单平台需求，性能最优，云开发原生集成，Skyline渲染引擎支持，无框架中间层损耗 |
| **后端方案** | 微信云开发（CloudBase） | 零运维、免域名备案、免鉴权获取用户身份、免费额度足够MVP阶段、与微信深度集成 |
| **数据库** | 云数据库（MongoDB文档型） | 随云开发内置，文档型适合灵活数据结构，无需建表，支持聚合查询 |
| **文件存储** | 云存储 | CDN加速、自动备案、与云数据库无缝配合 |
| **语音方案** | 预录制音频（edge-tts批量生成）+ 同声传译插件（可选） | 免费TTS工具批量生成300-500字音频存入云存储，支持个人主体；企业主体可启用同声传译插件处理动态内容 |
| **认证方案** | wx.login() + 云函数获取openid | 静默登录、无需用户手动授权、云函数自动获取openid |

### 2.2 技术约束

| 约束类型 | 具体限制 | 影响与应对 |
|---------|---------|-----------|
| **主包体积** | ≤2MB | 核心代码与首屏资源放主包，其余分包加载 |
| **总包体积** | ≤20MB | 汉字图片/音频放云存储，按需加载 |
| **云函数单次运行内存** | ≤256MB | 复杂统计分批处理 |
| **云函数并发数** | 基础版20 / 专业版100 | MVP阶段足够，需注意并发监控 |
| **云数据库同时连接数** | 基础版20 / 专业版100 | 高并发时通过云函数中转聚合 |
| **数据库读操作** | 基础版5万次/天 | 关键数据本地缓存，减少云端读取 |
| **数据库写操作** | 基础版3万次/天 | 进度数据批量提交，非实时写入 |
| **云存储容量** | 基础版5GB | 300-500字音频+图片约需1-2GB，足够 |
| **TTS频率限制** | 同声传译插件有调用频率限制（仅企业主体可用） | 主方案用预录制音频，不依赖TTS；同声传译仅作可选增强 |
| **TTS合成文本长度** | 同声传译插件 ≤50字符 | 单字/词级别合成，足够使用 |
| **基础库版本** | ≥2.10.0（基础功能） | 同声传译插件需≥2.14.0，启用时在app.json中配置 |
| **setData性能** | 单次数据量不宜过大 | 列表虚拟滚动、分页加载、避免频繁setData |
| **个人主体限制** | 同声传译插件不支持个人主体 | 个人主体使用预录制音频方案完全可用；同声传译插件仅企业主体可启用，用于动态内容TTS |

#### 3.2.3 状态管理

采用轻量级全局状态管理方案（基于小程序 globalData + 页面间事件通信）：

- **globalData**：存储全局共享数据（用户信息、登录态、总体进度摘要）
- **页面data**：页面级状态（当前关卡数据、游戏状态）
- **wx.setStorage**：持久化缓存（进度数据、解锁状态、设置项）
- **事件总线**：跨页面通信（自定义 EventEmitter，用于进度更新通知）

#### 3.3.3 离线策略

- **核心数据本地缓存**：汉字基础数据（characters.json）打包在小程序内，离线可用
- **进度离线记录**：闯关结果先写入本地Storage，联网时批量同步到云端
- **音频离线**：高频汉字音频首次播放后缓存到本地，后续离线播放
- **图片按需加载**：汉字图片从云存储加载，支持懒加载和CDN缓存

## 4. 数据库设计方案

### 4.1 数据库选型说明

**选择微信云数据库（MongoDB文档型数据库）**，理由：
- 随云开发内置，无需额外部署
- 文档型（JSON）数据结构灵活，适合教育游戏数据
- 支持聚合查询、索引优化
- 内置权限控制（集合级 + 记录级）
- 前端可直接读取（配合权限规则），也可通过云函数中转

## 5. API设计方向

### 5.1 核心API端点列表

所有API通过 `wx.cloud.callFunction()` 调用云函数实现。

#### 5.1.1 认证相关

| API名称 | 云函数 | 方法 | 入参 | 出参 | 说明 |
|---------|-------|------|------|------|------|
| 用户登录 | `login` | - | 无（云函数自动获取openid） | `{ userId, isNewUser, token }` | 静默登录，首次自动创建用户记录 |
| 更新用户信息 | `updateUser` | - | `{ nickname, avatar, birthDate }` | `{ success }` | 更新用户资料 |
| 获取用户信息 | `getUserInfo` | - | 无 | `{ user }` | 获取当前用户完整信息 |

#### 5.1.2 关卡与学习

| API名称 | 云函数 | 方法 | 入参 | 出参 | 说明 |
|---------|-------|------|------|------|------|
| 获取章节列表 | `getChapters` | - | 无 | `{ chapters: [...] }` | 获取所有章节及解锁状态 |
| 获取关卡列表 | `getLevels` | - | `{ chapterId }` | `{ levels: [...] }` | 获取指定章节的关卡列表 |
| 获取关卡详情 | `getLevelDetail` | - | `{ levelId }` | `{ level, characters: [...] }` | 获取关卡含汉字数据 |
| 提交关卡进度 | `submitProgress` | - | `{ levelId, stars, score, wrongAnswers, duration }` | `{ success, unlockedLevels, newAchievements }` | 提交闯关结果 |
| 获取学习进度 | `getProgress` | - | `{ chapterId? }` | `{ progress: [...], summary }` | 获取用户进度概览 |

#### 5.1.3 字卡与复习

| API名称 | 云函数 | 方法 | 入参 | 出参 | 说明 |
|---------|-------|------|------|------|------|
| 获取字卡数据 | `getCharacters` | - | `{ characterIds }` | `{ characters: [...] }` | 批量获取汉字数据 |
| 获取复习列表 | `getReviewList` | - | `{ count? }` | `{ characters: [...] }` | 基于遗忘曲线推荐复习汉字 |
| 记录学习行为 | `logLearnAction` | - | `{ characterId, action, result, context }` | `{ success }` | 记录单次学习行为 |

#### 5.1.4 成就与统计

| API名称 | 云函数 | 方法 | 入参 | 出参 | 说明 |
|---------|-------|------|------|------|------|
| 获取成就列表 | `getAchievements` | - | 无 | `{ achievements: [...], unlocked: [...] }` | 获取所有成就及解锁状态 |
| 获取学习统计 | `getLearnStats` | - | `{ dateRange? }` | `{ dailyStats, summary }` | 获取学习统计数据 |
| 获取学习报告 | `getReport` | - | `{ period }` | `{ report }` | 生成周/月学习报告 |

#### 5.1.5 数据同步

| API名称 | 云函数 | 方法 | 入参 | 出参 | 说明 |
|---------|-------|------|------|------|------|
| 上传进度 | `syncData` | upload | `{ progressData: [...] }` | `{ success, conflicts? }` | 离线数据批量上传 |
| 下载进度 | `syncData` | download | `{ lastSyncTime }` | `{ progressData: [...], achievements: [...] }` | 从云端恢复进度 |

#### 5.1.6 音频管理

| API名称 | 云函数 | 方法 | 入参 | 出参 | 说明 |
|---------|-------|------|------|------|------|

## 7. 不可行警告

### 7.1 明确不可行/受限的功能

| 功能 | 问题 | 严重度 | 替代方案 |
|------|------|-------|---------|
| **实时语音识别（儿童跟读评测）** | 微信小程序的录音API（wx.getRecorderManager）可录音，但端侧无法做语音评测；需对接第三方语音评测API（如驰声、先声），集成复杂且付费 | ⚠️ 中 | MVP阶段不做跟读评测，后续可集成第三方API |
| **手写汉字识别** | 小程序Canvas可绘制，但端侧OCR能力有限；需上传到云端识别，延迟较高 | ⚠️ 中 | MVP阶段用笔顺动画演示代替手写识别 |
| **复杂动画/骨骼动画** | 小程序对Lottie/DragonBones支持有限，复杂动画性能差 | ⚠️ 中 | 用CSS动画 + SVG + 序列帧替代 |
| **后台持续播放音频** | 小程序切后台后音频播放受限（除背景音频外） | ⚠️ 低 | 教育场景在前台使用即可 |
| **个人主体使用同声传译** | 同声传译插件不支持个人主体，无法用于动态TTS | 🟡 低 | 主方案使用预录制音频（edge-tts生成），个人主体完全可用；动态TTS为可选增强功能 |
| **实时多人对战** | 小程序WebSocket可行但云开发不原生支持长连接，需自建WebSocket服务 | ⚠️ 中 | MVP不做联机，后续可扩展 |
| **大数据量本地分析** | 端侧计算能力有限，复杂统计分析需在云函数中完成 | ⚠️ 低 | 统计逻辑放云函数 |
| **动态更新内容（免审核）** | 小程序不支持JS层动态下发执行，内容更新需重新发布 | ⚠️ 低 | 汉字数据放云数据库，可动态更新数据（非代码） |
| **3D效果** | 小程序WebGL性能受限，3D渲染效果差 | ⚠️ 低 | 用2.5D视差效果替代 |

### 7.2 需特别注意的技术风险

| 风险项 | 描述 | 应对措施 |
|-------|------|---------|
| **TTS频率限制** | 同声传译插件有调用频率限制（错误码-40001），且仅企业主体可用 | 主方案用预录制音频，不依赖TTS；同声传译仅用于企业主体的动态内容 |
| **TTS合成延迟** | 同声传译TTS需网络请求，首次播放有300-800ms延迟 | 预录制音频从云存储加载，首次下载后缓存到本地，后续播放无延迟 |
| **TTS音色限制** | 同声传译仅支持标准普通话；预录制方案使用edge-tts可选择多种音色 | edge-tts支持 zh-CN-XiaoxiaoNeural 等多种中文音色，可选用更亲和儿童的音色 |
| **云函数冷启动** | 云函数首次调用有200-500ms冷启动延迟 | 核心云函数配置预热；前端加loading过渡 |
| **数据库并发限制** | 基础版同时连接数仅20 | 前端不直连数据库，统一通过云函数中转 |
| **包体积超限** | 300-500字的图片+音频可能超包体积限制 | 图片/音频全部放云存储，本地仅放文字数据 |
| **setData性能** | 频繁setData导致卡顿（儿童操作频繁） | 合并setData、使用虚拟列表、避免大对象更新 |
| **Skyline兼容性** | Skyline渲染引擎部分组件不兼容 | MVP先用WebView渲染，待Skyline成熟后切换 |

## 8. 性能考量

### 8.1 性能目标

| 指标 | 目标值 | 测量方式 |
|------|-------|---------|
| 冷启动时间 | <1.5s | 微信开发者工具性能面板 |
| 页面切换 | <300ms | 手动测试 |
| 字卡点击响应 | <100ms | 手动测试 |
| 音频播放延迟 | <500ms（缓存命中）/ <1s（首次加载） | 手动测试 |
| 闯关加载时间 | <1s | 从点击关卡到可操作 |
| 帧率 | ≥50fps（动画场景） | 微信开发者工具帧率监控 |
| 云函数响应 | <800ms（含网络） | 云开发控制台监控 |

### 8.2 前端性能优化策略

#### 8.2.1 包体积优化
- **分包加载**：主包仅含首页+闯关核心，学习/成就模块分包
- **图片优化**：汉字图片使用WebP格式，尺寸控制在200x200px以内
- **按需加载**：非首屏组件延迟加载
- **Tree Shaking**：移除未使用的组件和工具函数

#### 8.2.2 渲染性能
- **虚拟列表**：汉字列表使用 `recycle-view` 或自定义虚拟滚动
- **合并setData**：避免频繁调用setData，合并批量更新
- **WXS增强**：频繁动画用WXS处理（如星星弹跳效果），减少JS-WebView通信
- **CSS动画优先**：能用CSS动画的不用JS动画
- **减少节点数**：WXML节点控制在1000以内/页面

#### 8.2.3 数据加载
- **本地缓存**：汉字基础数据打包在本地，无需网络请求
- **预加载**：当前关卡完成后预加载下一关数据
- **分页加载**：字典/列表页分页加载，每页20条
- **图片懒加载**：使用 `lazy-load` 属性

#### 8.2.4 音频优化
- **音频预加载**：页面加载时预创建InnerAudioContext实例
- **音频缓存**：首次下载后缓存到本地（wx.downloadFile + 本地路径），后续离线播放
- **音频合并**（可选）：多个短音频合并为一个文件，通过seek定位播放
- **预录制优先**：所有汉字发音均使用预录制音频（edge-tts生成），无需实时TTS
- **批量下载**：进入关卡时批量下载该关卡所有汉字音频到本地缓存
- **按需TTS（可选）**：仅企业主体可启用同声传译插件，用于未预录制的动态内容（如随机组词）

### 8.3 后端性能优化策略

#### 8.3.1 云函数优化
- **冷启动优化**：核心云函数保持实例温热（定时心跳）
- **批量操作**：进度提交、学习记录批量写入
- **数据聚合**：使用聚合查询替代多次查询
- **超时控制**：云函数超时设为5s（默认3s），避免复杂操作超时

#### 8.3.2 数据库优化
- **索引优化**：所有查询字段建索引（见4.3节）
- **缓存策略**：公共数据（汉字库、关卡数据）云函数内缓存
- **读写分离**：公共数据前端直读（权限设为所有人可读），私有数据走云函数
- **分页查询**：使用 `skip` + `limit` 分页，避免大结果集

#### 8.3.3 存储优化
- **CDN缓存**：云存储自带CDN，图片/音频自动加速
- **临时链接**：使用 `wx.cloud.getTempFileURL` 批量获取下载链接
- **文件命名**：按 `images/characters/{字}.webp`、`audio/characters/{字}.mp3` 规范命名

### 8.4 监控与告警

| 监控项 | 工具 | 告警阈值 |
|-------|------|---------|
| 云函数错误率 | 云开发控制台 | >5% |
| 云函数响应时间 | 云开发控制台 | >2s |

## 附录A：技术选型决策记录（ADR）

### ADR-001: 选择原生开发而非跨端框架

**背景**：项目仅需微信小程序单端，无跨端需求。

**决策**：选择微信小程序原生开发。

**理由**：
1. 性能最优，无框架中间层损耗，冷启动快15-20%
2. 云开发原生集成，API调用最直接
3. Skyline渲染引擎、自定义组件v3等新特性第一时间支持
4. 社区踩坑资料最丰富
5. 无跨端兼容问题，维护简单

**代价**：WXML/WXSS非标准Web语法，团队需学习微信特有开发模式。

### ADR-002: 选择云开发而非自建服务器

**背景**：MVP阶段需快速验证，团队规模小，无专职后端运维。

**决策**：选择微信云开发（CloudBase）。

**理由**：
1. 零运维，免域名备案，免HTTPS配置
2. 与微信登录态深度绑定，免鉴权获取openid
3. 免费额度足够MVP阶段（5GB存储、5万次/天读、3万次/天写）
4. 前端可直接操作数据库（配合权限规则），减少开发量
5. Serverless自动扩缩，无需容量规划

**代价**：强绑定微信云生态，后续迁移成本较高；云函数冷启动有延迟。

### ADR-003: 选择预录制音频方案而非实时TTS

**背景**：需汉字发音功能，有实时TTS（同声传译插件）和预录制音频两种方案。用户暂不具备企业主体，同声传译插件不支持个人主体。

**决策**：主方案采用预录制音频（edge-tts批量生成300-500字+常用词组发音，存入云存储），同声传译插件降为可选增强（仅企业主体启用，用于动态内容TTS）。

**理由**：
1. edge-tts免费开源，可批量生成高质量中文语音
2. 预录制音频支持个人主体小程序，无主体限制
3. 预录制音频从云存储加载，首次下载后缓存本地，播放无延迟
4. edge-tts支持多种中文音色（如XiaoxiaoNeural），可选择更亲和儿童的音色
5. 同声传译插件作为可选增强，企业主体时可用于未预录制的动态内容
6. 消除TTS频率限制和合成延迟问题

**代价**：需在开发阶段离线生成并上传音频文件；动态内容（如随机组词）发音需预录制覆盖或依赖可选TTS。
