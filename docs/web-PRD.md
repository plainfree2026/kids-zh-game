# 网页版识字游戏 · PRD（产品需求文档）

> 版本：v0.1（待用户确认）
> 日期：2026-07-18
> 形态：单个 HTML 文件，双击即玩，零安装零后端

---

## 1. 产品定义

**一句话**：面向 6 岁儿童的网页版识字产品，包含「识字测评」+「抓字小游戏」两块，覆盖中文常用 3000 字，核心诉求是**有趣**。

- **目标用户**：6 岁儿童（主要操作者）；家长（陪玩 / 看测评结果）
- **使用场景**：手机浏览器为主（家中平板 / 手机），碎片化时间 5–15 分钟一局
- **核心问题**：市面识字产品多为 App、且重付费；家长想要一个**打开就能玩、不装不买、孩子觉得好玩**的识字工具

## 2. 与小程序版的关系

这是**独立的一条产品线**（网页版），不复用小程序代码。内容池同为常用字，但交互与形态完全不同。小程序版（已交付）继续独立存在。

## 4. 用户流程

```
首页(字宝 + [开始测评] [开始游戏])
   │
   ├─ 开始测评 → 测评页(听音4选1 ×20) → 报告页(识字量≈X, 建议从第Y级开始) → [去玩游戏]
   │
   └─ 开始游戏 → 选关/自动按进度 → 抓字游戏页(下落+点选) → 结算页(得分/星星/连击) → 重玩/回首页
```

## 5. 关键交互设计

### 5.1 测评（听音选字）
- 每题：自动播放一个字的拼音发音（如 "mù"），屏幕出现 4 个汉字候选（1 正确 + 3 干扰）
- 点正确：绿光 + 鼓励音；点错误：红框 + 该字正确读音再播一次
- 进度条显示 1/20；结束出报告：估算识字量（按正确率映射）、建议起始级别、可「重新开始」或「去玩游戏」

### 5.2 抓字游戏
- 顶部持续下落汉字（每次 1–3 个），同时播放其中一个的发音
- 底部 4 个候选字（含当前目标字）；点中正确字 = 得分 + 连击 +1
- 漏接（字落到底）或点错 = 扣 1 生命；生命归零 = 本局结束
- 连击特效：连击数越高，字下落越快、特效越炫（果冻弹跳 + 星星轨迹）
- 每局约 60 秒或按关卡字数结束，结算给星星

## 6. 验收标准（QA 依据）

| 编号 | 功能 | Given | When | Then |
|------|------|-------|------|------|
| AC01 | 测评发音 | 进入测评页 | 每题加载 | 自动用中文朗读目标字拼音 |
| AC02 | 测评答题 | 显示 4 候选 | 点正确字 | 计 1 分并播鼓励音；错误则提示正确读音 |
| AC03 | 测评报告 | 20 题答完 | 结束 | 显示识字量估算 + 建议级别 + 两个入口 |
| AC04 | 抓字下落 | 进入游戏 | 开始 | 汉字从顶部持续下落，有发音引导 |
| AC05 | 抓字点选 | 目标字在候选 | 点中正确 | 得分 + 连击 +1，特效触发 |
| AC06 | 抓字失败 | 漏接/点错 | 生命-1 | 生命归零则本局结束进结算 |
| AC07 | 发音可用 | 任何汉字 | 触发朗读 | 手机浏览器用 speechSynthesis 出中文音（无音频文件） |
| AC08 | 进度留存 | 玩过之后 | 刷新页面 | localStorage 恢复星星/最高分/已识字数 |

## 7. 内容数据

- 来源：《通用规范汉字表》一级常用字（开源数据 `togscc` / `chinese-dictionary`）
- 规模：3000 字，带拼音；按频率/笔画分 6 级（每级 500）
- 多音字：取最常用音（测评/游戏只考常用音，避免歧义）
- 构建：用 `pinyin-pro` 在构建期生成 `char→pinyin` 映射，内联进单 HTML，保证离线可用

## 8. 约束

- 仅前端，无网络请求（除可选的 CDN 字体/图标）
- 移动优先响应式（手机竖屏为主，平板/桌面自适应）
- 单文件 ≤ 合理体积（3000 字词典内联约 100–200KB，可接受）
- iOS Safari 限制：首次发音需用户手势触发（首页「开始」按钮即手势，已规避）
