# 网页版识字游戏 · Architecture（技术架构文档）

> 版本：v0.1（待用户确认）
> 日期：2026-07-18

---

## 1. 技术选型（对比矩阵）

| 维度 | 方案 A：单 HTML + 原生 JS（**选用**） | 方案 B：React/Vue 工程 | 方案 C：小程序迁移 |
|------|------|------|------|
| 交付形态 | 1 个 .html，双击即玩 | 需构建+部署 | 不适用 web |
| 离线可用 | ✅（数据内联） | ❌ 需服务器 | — |
| 后端依赖 | 无 | 无（纯前端也可） | 需云开发 |
| 趣味性开发 | Canvas/CSS 动画足够 | 同左，但更重 | — |
| 适合 6 岁即玩 | ✅ 最强 | 一般 | — |

**结论**：选方案 A。完全契合用户「单个 HTML 文件、双击即玩、有趣」的诉求。

## 2. 系统架构（分层）

```
┌──────────────────────────────────────────┐
│               index.html（单文件）            │
├────────────┬────────────┬──────────────────┤
│  数据层     │  音频层     │  游戏/测评引擎     │
│  CHAR_DICT  │ speechTTS() │  GameLoop / Quiz  │
│ (3000字内联) │ (Web Speech)│  (requestAnimationFrame)│
├────────────┴────────────┼──────────────────┤
│  UI/动画层   │  进度层                       │
│  CSS动效/Canvas│ localStorage(星星/最高分)    │
└──────────────┴─────────────────────────────┘
```

## 3. 关键技术决策

### 3.1 拼音与数据 —— 内联词典，离线可用
- 构建期用 `pinyin-pro` 对 3000 字生成 `{char: pinyin, tier: n}` 映射
- 以 `const CHAR_DICT = [...]` 内联进 HTML（约 100–200KB）
- 运行时直接查表，无需网络、无需 CDN
- 分级：`tier = 1..6`，按字频/笔画粗分，每级 500 字

### 3.2 发音 —— Web Speech API
- `speechSynthesis.speak(new SpeechSynthesisUtterance(text))`，`lang='zh-CN'`
- **无需 3000 个音频文件**，任意字即时朗读
- 触发时机：用户手势内调用（首页「开始」按钮即首次手势，规避 iOS 限制）
- 每次播放前 `cancel()` 防堆叠；`rate=0.8` 儿童友好语速
- 降级：若浏览器不支持，显示拼音文字兜底
- ⚠️ 多音字只读常用音；测评不考多音歧义

### 3.3 抓字游戏引擎
- 游戏循环：`requestAnimationFrame` 驱动下落位置
- 渲染：DOM 元素（`.char-tile`）+ CSS `transform: translateY` 过渡，比 Canvas 更易做圆角/动画且性能足够 6 岁场景
- 候选区：底部 4 个 `.option` 按钮（含 1 目标字 + 3 同 tier 干扰字）
- 碰撞/判定：字落到底部线 = 漏接；点候选 = 判定
- 难度曲线：连击越高 → 下落速度 +1、同屏字数 +1

### 3.4 进度存储
- `localStorage` 键：`shizi_stars`, `shizi_best`, `shizi_known`, `shizi_tier`
- 纯前端，无同步；换设备不保留（MVP 可接受）

### 3.5 响应式
- 竖屏手机优先（`max-width: 480px` 容器居中）
- 平板/桌面：容器限宽居中，触屏与鼠标事件兼用（`click`/`touchend` 统一）

## 4. 文件结构（单文件内部分区）

```
index.html
├─ <style>  设计 Token + 布局 + 动效
├─ <body>   首页 / 测评 / 报告 / 游戏 / 结算 五个视图（display 切换）
└─ <script>
   ├─ CHAR_DICT（3000 字内联）
   ├─ speechTTS()       发音封装
   ├─ Quiz 模块         测评逻辑
   ├─ Game 模块         抓字循环 + 判定
   ├─ Storage 模块      进度
   └─ UI 模块           视图切换 + 动效(Confetti/连击)
```

> 构建期内联词典：开发时用 `pinyin-pro` 生成 `dict.js`，再由脚本拼进 `index.html`；最终交付物是**单个 index.html**。

## 5. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| iOS Safari 首次无声音 | 测评/游戏开局静音 | 首页「开始」按钮即用户手势，预触发一次空 speak 激活语音引擎 |
| 部分安卓浏览器无 zh-CN 语音 | 听音失效 | 显示拼音文字兜底 + 提示换浏览器 |
| 多音字歧义 | 孩子困惑 | 只考常用音；干扰项同 tier 取不同音 |
| 单文件体积 | 加载略慢 | 3000 字压缩内联 ≈150KB，gzip 后更小，可接受 |
| 触屏误点 | 错答 | 候选按钮加大（≥64px）、间距充足 |

## 6. 不在本期范围

- 服务端、账号、支付、家长后台、笔顺描摹、真人语音评测
